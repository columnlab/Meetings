GitHub Basics

Git is a version control system for tracking changes

GitHub is a social coding platform using Git but also supporting collaborations

GitHub Vocab

A repository ("repo") is project-level collection of files

Users "push" and "pull" changes to the project

Copy ("fork") any repo to make your own changes

Collisions may require merging or reverting

Individual accounts + organizations + projects

Accessing GitHub Page

In the Settings for a repo, go to "Pages", and select the "main" branch under "Source"

Then you should be able to access the website at accountname.github.io/reponame/

Create a repository called accountname.github.io to host a website

Basic Web Design

You'll need a index.html as a landing page

Otherwise link directly into the HTML file

There's not much at https://columnlab.github.io/Meetings/

This presentation is an HTML file located at https://columnlab.github.io/Meetings/2021/2021-05-19-Group.html

Workflow

Make the repo and enable the "Pages" option

Clone the repo to your local machine

Make/open an HTML file inside the repo directory

Save your HTML file, use GitHub Desktop to "commit to main", and then "push" to origin

Go to your account/org github.io page to see it!

It may take a minute or so for your changes to update on the page

Reveal.js 📜

A web framework for interactive presentations

But slick and lightweight way to share presentations!

Setting up Reveal.js with a CDN

The Reveal.js installation documentation recommends a "local hosting" setup

A local hosting setup requires you to maintain and serve the Reveal.js files yourself

Setting up Reveal.js with a CDN

Content Delivery Networks (CDNs) host data like videos and software instead of your server

In addition to laziness and simplicity, this has benefits for speed, consistency, and security

Setting up Reveal.js with a CDN
